FAQ

商家申请前最常问的问题

PayPal、Stripe、Creem 之前直接拒了我，理由都没给。你们会不会一样？

AI 工具、虚拟商品、社群类业务经常被这样拒——风控系统很多时候扫一眼行业分类就直接毙掉，根本没人看你实际卖什么。我们是人工审核：产品本身、收款流程、定价、退款政策、交付方式，逐项看。这不代表一定通过，但至少是看你的真实业务做决定，不是看一个分类代码。

开通之前你们具体会看什么？

网站、收银流程、目标市场、预计交易量、支付方式、客服流程，还有争议风险。这些本来就是任何支付渠道该了解的信息，我们只是选择提前问清楚，而不是等拒付率飙升了才回头追问。

我做的是 AI 生图或 AI 写作类产品，是不是天然高风险？

不会自动判高风险。AI 类产品本来就不太吃标准行业分类，这恰恰是我们专门处理的地带。我们看交付方式、退款政策、用户实际怎么用这个产品，而不是看到"AI"两个字就直接拒。

审核要多久？

看业务复杂度。一个标准的 SaaS 订阅，通常比跨多个市场的社群收款方案审得快。申请材料里有需要补的地方，我们会提前说，不会让你干等一个不会来的结果。

审核通过是不是就结束了，给个账户就完事？

不是，通过只是开始。上线之后，我们支持托管收银台、支付链接、Shopify 和 WooCommerce 集成、REST API 自定义接入、AI 用量计费、社群收款、多币种结算，还有拒付预警。

我用 Shopify 或 WooCommerce，是不是要重新搭建收银流程？

不用重建。已经在用 Shopify 或 WooCommerce？我们直接对接现有系统。需要更定制的方案——比如 AI 产品的按量计费，或者 Telegram、Discord 社群里的收款？REST API 和支付链接也支持。

我的业务对不上标准的行业分类，这算问题吗？

这恰恰是我们主要服务的对象。行业分类本来就没为 AI 用量计费、付费 Telegram 或 Discord 社群、跨境订阅制产品设计过，这也是很多申请被标准支付渠道直接拒绝的原因。我们看的是业务实际在做什么，而不是它该套进哪个分类。

这是自助开通，还是有真人跟进？

有真人跟进，不是工单排队。审核你申请的运营人员，会一路跟到上线、结算、争议处理，以及之后任何账户变动。

如果之后拒付率上升，或者情况有变化，账户会不会直接被冻结？

我们不能承诺情况永远不变——支付风险本身就是动态的。但我们能保证沟通方式：拒付率上升或有需要处理的问题，你会先从真人那听到消息，有时间应对，而不是账户突然被冻结了才发现。
